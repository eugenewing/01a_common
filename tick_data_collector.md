#### 1. Назначение и функционал
мы создаем проект для сбора тиковых данных с терминала MetaTrader 4. папка проекта F:\Projects\tick_data_collector. Советник собирает тиковые данные в оперативную память, передавая их в TickSenderApi.dll. TickSenderApi.dll асинхронно, параллельно с работой советника отправляет эти данные на сервер. сервер передает данные в базу данных и возвращает ответ.
#### 2. Стек технологий и версии
Стек технологий: Советник на mql4, TickSenderApi.dll на C++, сервер на Java, база данных на PostgreSQL. Версия Java: 25. Архитектурный подход: чистое разделение. Инструменты: без Spring Boot, чистый java.
#### 3. Формат взаимодействия
Протокол: REST API (HTTP). Формат данных: JSON
#### 4. Границы и ограничения проекта
Безопасность: сейчас делаем без защиты и авторизации. Масштаб: информационная система будет развернута на одном ПК. UI клиента: не будет.
#### 5. Инструменты сборки и структура
Сборщик: Maven. Структура репозитория: multi-module проект.

#### 111. Принцип работы и взаимодействия советника и dll
советник накапливает определенное количество элементов массива структур тиковых данных, добавляет их в dll в массив передачи и обнуляет свой массив. асинхронная функция отправки данных внутри dll сканирует длину массива передачи и если его длина достигает определенного количества, отправляет пакет данных на сервер и ждет ответ. в случае если сервер не принял данные, то функция отдает детализацию ошибки в советник. если сервер принял данные, то функция удаляет удачно оправленную часть массива передачи. переменные для длины массива в советнике и длины массива передачи в dll должны определяться в настройках советника.
#### 111. Принцип работы сервера
подключение к базе данных на сервере должно осуществляться с использованием HicariCP. настройки подключения к базе данных должны содержаться во внешнем ini файле. для получения настроек из ini нужно будет установить зависимость ini4j. на сервере должны работать классы web listener и servlet. при запуске сервера web listener устанавливает подключение к базе данных через HicariCP по настройкам из ini и записывает во внешний лог файл статус запуска сервера (удачно/не удачно). для записи во внешний лог файл будем использовать java.util.logging. входящие запросы будет обслуживать только один сервлет.
