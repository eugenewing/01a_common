#### 1. Назначение и функционал
мы пишем информационную систему (далее ИС) для контроля материальных ценностей в организации. приложение называется ISkRa, папка проекта F:\Projects\iskra. суть ИС в следующем. есть предметы и помещения. ИС должна давать возможность удобно и быстро перемещать различные предметы в разные помещения и давать возможность удобно просматривать предметы в помещениях. помимо этого нужно сделать функционал опциональной привязки предмета к предмету, например для того, чтобы связать предмет компьютер и предмет монитор. у каждого предмета обязательно должно быть определено материально-ответственное лицо (МОЛ). также у ИС должна быть поддержка вывода печатной формы материально-технического оснащения помещения.
2. Стек технологий и версии
Версия Java - 25. Архитектурный подход - чистое разделение. Инструменты - чистый java без Spring Boot. База данных - PostgreSQL. Фронт - Angular + Angular material.
3. Формат взаимодействия
Протокол - REST API (HTTP). Формат данных - JSON.
4. Границы и ограничения проекта
Авторизация - Basic Auth. Масштаб - ИС будет ограничена работой на одном серверном компьютере. UI клиента - веб-страница.
5. Инструменты сборки и структура
Сборщик - Maven. Структура репозитория - multi-module проект.
6. Прочая информация по проекту
проект подготовлен под разработку и сборку в IntelliJ Idea с использованием maven. исходный код сервера будет расположен по пути iskra\src\main\java\ru\eugenewing. исходный код фронта будет расположен по пути iskra\client\src\app. ИС будет использоваться в локальной сети организации. использование ИС будет осуществляться пользователями как через ПК, так и через смартфоны. mvp классы бекенда: servlet (прослушивание запросов с фронта), web listener (начальная инициализации сервера при запуске/перезапуске). для подключения к субд на бекенде будет использоваться HikariCP. ИС будет запускаться на другом ПК (сервере) в среде XAMPP как скомпелированный .war файл. необходимо добавить простое логирование во внешний файл на старте сервера через web listener (инициализирован успешно или с ошибкой) и простой файл конфига для сервера. для логирования и конфига должны быть определены зависимости в pom.xml. утвержденная диаграмма базы данных по реляционной модели уже есть в папке docs (файл er_diagram.mmd). ты пишешь исходный код бекенда и фронта в вышеописанных папках. компиляциями, тестами и деплоями буду заниматься я. ближайшим действием ты должна будешь создать файл контекста проекта PROJECT_CONTEXT.md и файл задач к выполнению NEXT_STEPS.md в корне проекта.
