ты главная full-stack разработчица. рассказываю тебе о проекте, который мы будем разрабатывать.
#### 1. Назначение и функционал
мы создаем проект для сбора тиковых данных с терминала MetaTrader 4. папка проекта F:\Projects\tick_data_collector. Советник собирает тиковые данные в оперативную память, передавая их в TickSenderApi.dll. TickSenderApi.dll асинхронно, параллельно с работой советника отправляет эти данные на сервер. сервер передает данные в базу данных и возвращает ответ.
#### 2. Стек технологий и версии
Стек технологий: Советник на mql4, TickSenderApi.dll на C++, сервер на Java, база данных на PostgreSQL. Версия Java: 25. Архитектурный подход: чистое разделение. Инструменты: без Spring Boot, чистый java.
#### 3. Формат взаимодействия
Протокол: REST API (HTTP). Формат данных: JSON
#### 4. Границы и ограничения проекта
Безопасность: сейчас делаем без защиты и авторизации. Масштаб: информационная система будет развернута на одном ПК. UI клиента: не будет.
#### 5. Инструменты сборки и структура
Сборщик: Maven. Структура репозитория: multi-module проект.
#### 6. Структура папок в проекте (папки уже созданы)
Советник: TickDataCollector\
TickSenderApi.dll: TickSenderApi\
Сервер: src\main\java\ru\eugenewing\
#### 7. Прочая информация по проекту
сервер будет разворачиваться как скомпелированный файл war в среде XAMPP на другом ПК. ты должна будешь писать исходный код советника, TickSenderApi.dll и сервера в папках проекта, обозначенных выше. я буду проверять твой код, проводить тесты и компиляцию советника, dll и сервера. создание и настройку базы данных я также беру на себя. принцип работы и взаимодействия советника и TickSenderApi.dll следующий. советник накапливает определенное количество элементов массива структур тиковых данных, добавляет их в dll в массив передачи и обнуляет свой массив. асинхронная функция отправки данных внутри dll сканирует длину массива передачи и если его длина достигает определенного количества, отправляет пакет данных на сервер и ждет ответ. переменные для длины массива в советнике и длины массива передачи в dll должны определяться в настройках советника. подключение к базе данных на сервере должно осуществляться с использованием HicariCP. настройки подключения к базе данных должны содержаться во внешнем ini файле. для получения настроек из ini нужно будет установить зависимость ini4j. на сервере должны работать классы web listener и servlet. при запуске сервера web listener устанавливает подключение к базе данных через HicariCP по настройкам из ini и записывает во внешний лог файл статус запуска сервера (удачно/не удачно). для записи во внешний лог файл будем использовать java.util.logging. входящие запросы будет обслуживать только один сервлет.
