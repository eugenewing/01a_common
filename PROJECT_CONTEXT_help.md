### Вот 5 основных блоков информации, которые нужно передать модели перед стартом:
1. Назначение и функционал (Что делаем?)
	- Суть проекта: Опишите кратко, для чего нужно приложение.
	- Главные фичи: Перечислите 3–5 ключевых функций.
2. Стек технологий и версии (На чем делаем?)
	- Версия Java: Укажите конкретную версию (например, Java 17, Java 21). От этого зависит синтаксис.
	- Архитектурный подход: Будет ли это монолит или чистое разделение.
	- Инструменты: Нужен ли Spring Boot, или приложение должно быть "pure Java" (на чистых сокетах / HttpServer), чтобы не перегружать проект.
	- База данных: Нужна ли БД (H2, PostgreSQL) или пока достаточно хранения в оперативной памяти(InMemory/ConcurrentHashMap).
3. Формат взаимодействия (Как общаются клиент и сервер?)
	- Протокол: REST API (HTTP), gRPC, WebSockets или чистые TCP/UDP сокеты.
	- Формат данных: JSON, XML или сериализация объектов Java.
4. Границы и ограничения проекта (Чего мы НЕ делаем?)
	- Безопасность: Нужна ли авторизация (JWT, Basic Auth) или сейчас делаем без защиты.
	- Масштаб: Ограничено ли приложение работой на одном компьютере (localhost).
	- UI клиента: Будет ли это консольный интерфейс (CLI), графический (JavaFX/Swing) или веб-страница.
5. Инструменты сборки и структура
	- Сборщик: Maven или Gradle.
	- Структура репозитория: Это будет multi-module проект (один parent-проект, внутри модули client и server) или два абсолютно разных независимых проекта.
